OTTIMIZZAZIONE DATABASE

Con la proprietÃ  self::$_enable_cache Ã¨ possibile caricare i risultati delle query (non di struttura) in un array in modo da evitare l'esecuzione multipla della stessa query.
Le query che vengono caricate nell'array sono quelle che passano dal metodo selectquery(), e non riguardano quindi le query di struttura, quali quelle presenti nei metodi:

  • fieldInformations()
  • getTableStructure()
  • getFieldsName()

In particolari query gestite dal metodo selectquery() Ã¨ talvolta necessario non salvarne i risultati (vedi autoIncValue()). In questo caso Ã¨ suffuciente passare a selectquery() il parametro cache=false.

I risultati delle query sulla struttura delle tabelle (getTableStructure()) vengono gestiti dalla cache di gino (Principali impostazioni di sistema) e salvati su file (vedi Model::structure()).
